#MSA

# 1. 서비스 통신 및 인터페이스 패턴
- API Gateway 패턴 : 클라이언트가 단일 진입점(Gateway)를 통해 통신, 인증, 권한부여, 부하 분산을 중앙에서 처리 하는 방식
- Service Discovery 패턴: 서비스의 위치(IP, 포트)가 동적으로 변하는 환경에서 서비스들이 서로를 자동으로 찾을 수 있게 해주는 방식

# 2. 데이터 관리 패턴 : '데이터의 일관성' 유지
* Database per Service : 각 서비스는 자신만의 전용 데이터베이스를 가짐. 서비스간 결합도는 낮추지만, 데이터 조인이 어렵다는 단점이 있음
* **Saga 패턴** : 여러 서비스에 걸친 분산 트랜잭션을 관리, 한 서비스에서 작업이 실패하면 이를 취소하는 **보상 트랜잭션**을 실행하여 데이터 일관성을 맞춤
* CQRS(Command Query Responsibility Segregation) : 명령(데이터 변경)과 조회(데이터 읽기)의 모델을 분리하여 성능과 확장성을 최적화

# 3.회복성과 안정성 패턴 : 특정 서비스에 장애가 발생했을때 전체 시스템으로 번지는 것을 차단
* Circuit Breaker : 특정 서비스에 반복적으로 에러가 발생하면 호출을 즉시 차단하여 시스템 전체가 마비되는 것을 방지
* Bulkhead 패턴: 배의 격벽처럼 서비스를 그룹화하여 한 구역의 장애가 다른 구역으로 전염되지 않도록 격리

# 4. 관찰 가능성(Observability) 패턴
* Log Aggregation : 여러 서비스의 로그를 한 곳에 모아 관리(예:ELK 스택)
* Distirubted Tracing : 클라이언틔 요청이 여러 서비스를 걸치 때, 전체 흐름을 추적하여 어디서 병목이나 에러가 생기는지 파악(예: Jaeger, Zipkin)
