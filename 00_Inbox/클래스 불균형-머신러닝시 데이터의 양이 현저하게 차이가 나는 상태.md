#클래스불균형

데이터 셋이 다음과 같이 구성되어 있다고 가정할 때
- 다수 클래스(Majority Class) : 데이터의 양이 압도적으로 많은 그룹(정상 데이터: 99%)
- 소수 클래스(Minority Class) : 데이터의 양이 매우 적은 그룹(사기 거래 데이터 : 1%)

# 발생하는 주요 사례
- 금융 사기 탐지 : 대부분은 정상 결제이며, 카드 도난이나 부정 사용은 극히 일부분
- 질병 진단 : 특정 희귀병 검사 데이터에서 양성 반응은 음성 반응에 비해 현저히 적음
- 제조 공정 불량 탐지 : 공정 시스템이 우수할 수록 불량품 데이터는 정상 제품보다 훨씬 적게 발생

# 해결 방법

| 구분     | 방법                    | 설명                                |
| ------ | --------------------- | --------------------------------- |
| 데이터 기반 | 언더 샘플링(Undersampling) | 다수 클래스의 데이터를 줄요서 비율을 맞춤           |
|        | 오버 샘플링(Oversampling)  | 소수 클래스의 데이터를 복제하거나 생성(SMOTE)하여 늘림 |
