#text_segmentation

* ## 텍스트 분할(text segmentation)
	* 연속된 긴 텍스트를 의미 있는 물리적 또는 논리적 단위(문장, 문단, 주제 등)로 나누는 과정
	* [분할 단위와 종류]
		* 문장 분할(Sentence Segmentation)
			* 텍스트를 개별 문장 단위로 나누는 작업
			* 주로 마침표, 물음표, 느낌표를 기준으로 하지만 'U.S.A' 나 'Mr.Kim' 처럼 마침표가 포함된 약어 등을 구분해내는 정교함이 필요
		* 단어/토큰 분할(Word Segmentation)
			* 단어 단위로 나누는 작업
		* 주제 분할(Topic/Document Segmentation)
			* 긴 문서 내에서 주제가 바뀌는 지점을 찾아내는 과정
			* 한 문서 안에서 서론, 본론, 결론을 나누거나 뉴스기사에서 서로 다른 사건을 다루는 부분을 분리
			* 알고리즘 예시 : #TextTiling (단어의 빈도 변화를 감지해 주제 전환점 포착)


* ### [최신 NLP(RAG)에서의 텍스트 분할 전략]

| 전략                            | 설명                                          | 장점/단점                        |
| ----------------------------- | ------------------------------------------- | ---------------------------- |
| Fixed-size Chunking           | 단순히 글자 수 나 토큰 수(예: 500자)로 나눔                | 구현이 매우 쉬움<br>문장이 잘려 의미 훼손 가능 |
| Recursive Character Splitting | 문단->문장->단어 순서로 의미 단위를 유지하며 자름               | 문맥 보존 능력이 뛰어나 가장 많이 사용됨      |
| Semantic Chunking             | 임베딩(Embedding)을 통해 의미적 유사도를 측정하고 주제가 바뀔때 자름 | 가장 지능적이지만 연산 비용이 높음          |
