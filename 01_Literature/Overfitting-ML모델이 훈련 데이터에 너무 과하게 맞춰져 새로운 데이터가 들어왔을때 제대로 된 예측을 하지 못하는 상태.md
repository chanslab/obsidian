#Overfitting 

"시험공부를 문제의 원리를 이해하는게 아니라 기출문제의 정답과 글자 토씨 하나까지 통째로 외워버린 상황"

# 특징
1. 훈련 데이터에서는 높은 성능
2. 실제 데이터에서는 낮은 성능

# 왜? 발생하는가?
- 데이터 부족
	- 모델이 일반적인 패턴을 찾기에 학습 데이터 양이 너무 적을 때 발생
- 모델의 과한 복잡도
	- 풀려는 문제에 비해 모델의 층(Layer)이 너무 깊거나 파라미터가 너무 많을 때 발생
- 학습 회수([^1]Epoch) 과다 
	- 너무 오랫동안 반복해서 학습시키면 데이터의 미세한 노이즈까지 학습
	- 반면에 너무 학습이 적으면 성능이 떨어짐(과소적합, Underfitting)


[^1]: 에포크(Epoch)는 머신러닝에서 학습 데이터 전체를 모델이 한번 훑었을 때의 단위
	훈련 데이터가 1000개 있다고 가정할 때, 1000개의 데이터를 모두 사용하여 학습을 한번 완료하면 1에포크가 지났다라고 말함
	
	- 에포크(Epoch) : 데이터 전체를 한 번 다 본 상태
	- 배치 크기(Batch size) : 한번에 학습될 데이터의 묶음 단위
	- 이터레이션(Iteration) : 한 에포크를 마치기 위해 필요한 학습 횟수

# 어떻게 해결하는 가?


| 해결기법                      | 설명                                               |
| ------------------------- | ------------------------------------------------ |
| 데이터 증강(Data Augmentation) | 데이터를 변형(이미지 회전, 대칭 등) 하거나 추가하여 데이터 양을 늘림         |
| 드롭아웃(Dropout)             | 학습 과정에서 신경망의 일부 노드를 무작위로 꺼서 특정 노드에 <br>의존하지 않게 함 |
| 조기 종료(Early Stopping)     | 검증 오차가 증가하는 시점에 학습을 멈춤                           |
| 규제(Regularization)        | 모델의 가중치가 너무 커지지 않도록 패널티를 부여(L1, L2)              |
