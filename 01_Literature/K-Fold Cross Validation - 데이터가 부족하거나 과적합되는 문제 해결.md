#ML #Overfitting

머신러닝에서 데이터가 부족하거나 특정 데이터셋에만 과적합(Overfitting)되는 문제를 해결하기 위해 사용하는 대표적인 방법

# K-폴드 교차 검증이란?
전체 데이터를 K개의 일정한 크기인 '폴드(Fold)'로 나누고, 각 폴드를 한번씩 테스트 데이터로 사용하고 나머지를 학습 데이터로 사용하는 과정을 K번 반복하는 방법

## 작동 프로세스
1. 전체 데이터를 서로 겹치지 않게 K개의 그룹으로 나눔
2. 첫번째 그룹을 **검증 데이터(Validation Set)**로, 나머지 k-1 개 그룹을 '학습데이터(Training Set)'으로 설정하여 모델을 학습**
3. 학습된 모델을 검증 데이터로 평가하여 평가지표(예: Accuracy)를 기록
4. 이 과정을 검증 데이터 그룹을 바꿔가면서 K번 반복
5. 최종적으로 K개의 평가 점수를 평균 내어 모델의 성능을 확정

## 왜? 사용하는가?
- 데이터 활용 극대화
	- 모든 데이터를 학습과 평가에 사용하므로 데이터가 적을 때 유용
- 일반화 성능 파악
	- 특정 데이터셋에만 잘 작동하는게 아닌, 전체적인 모델의 안정성을 확인 가능
- 평가의 신뢰도
	- K번의 평균값이므로 객관적

반면에 계산 비용이 증가하는 단점도 있음


# 데이터의 특성에 따라 나누는 기법이 다름
- Stratified K-Fold
	- 분류 문제에서 타겟 레이블의 비율이 불균형 할 때
		- 예) 암환자 1%, 정상 99%
		- 각 폴드에도 이 비율이 유지되도록 나누는 방식
- Repeated K-Fold
	- K-폴드 과정을 여러 번 반복하여 평가의 분산을 더 줄이는 방식
