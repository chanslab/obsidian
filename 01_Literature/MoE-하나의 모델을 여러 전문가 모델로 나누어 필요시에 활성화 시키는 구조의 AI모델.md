#MoE 

하나의 거대한 모델을 여러 '전문가(Expert) 모델'로 나누어 놓고 입력마다 필요한 전문가를 선택적으로 화성화 시키는 구조의 AI모델

LLM은 점점 커지고 있는데 모든 파라미터를 매번 계산하면 
- GPU 비용 증가
- 속도 저하
- 전력 소모 증가
의 문제가 발생


# MOE (Mixture of Expert)의 구조
1. Experts : 전문가 네트워크
	1. 여러개의 작은 모델 집합
2. Gating Network : 어떤 모델을 사용할지 선택,  라우팅 역할
	1. [^1]Softmax 등을 사용해 확률적으로 전문가를 선택
	2. 보통 Top-1 , Top-2 전문가만 활성화
3. Sparse Activation
	1. 전문가 수가 100개여도 실제로는 1~2개만 계산에 참여하여 비용 절감

[^1]: 딥러닝에서 주로 다중 클래스 분류(Multi-class Classification) 문제를 해결할 때 사용하는 활성화 함수(Activation Function)
	- 여러 개의 선택지 중에 어디에 속할 확률이 가장 높은지 계산하여 총합이 1(100%)이 되도록 결과를 내보내는 함수

# 특징
1. 비용 절감(Sparse 계산)
	1. 전체 파라미터는 매우 크지만 실제 계산은 일부만 사용
	2. 큰 모델처럼 똑똑하면서도 계산량은 작은 모델 수준
2. 확장성(Scalability)
	1. 전문가 수를 늘려도 비용은 크게 증가하지 않음
	2. 매우 큰 모델을 만들 수 있음
3. 전문성 향상
	1. 각 전문가 모델이 특정 패턴에 자연스럽게 특화됨
