#Cot #ReAct #ToT #프롬프트엔지니어링 #ChainOfThought 

1. Cot (Chain of Thought, 생각의 사슬)
	1. "단계별로 생각해 보자"
	2. 문제를 작은 단위로 쪼개 논리적인 흐름을 따라 답변을 생성
	3. (장점) 복잡한 산술, 상식적 추론 문제에서 정확도가 비약적으로 상승
	4. (단점) 모델 내부의 지식에만 의존하므로, 잘못된 정보를 사실처럼 말하는 환각 현상이 발생할 수 있음
2. ReAct (Reasoning + Acting, 추론과 행동)
	1. "무엇을 모르는지 " 판단하고 외부 정보(검색 엔진, API등)를 가져와 보완
	2. 추론(Thought) -> 행동(Action) -> 관찰(Observation)의 루프를 반복
	3. (한계) 외부 도구 사용에 따른 비용과 시간이 추가되고, 루프가 길어지면 비효율적일 수 있음
3. ToT (Tree of Thoughts, 생각의 나무)
	1. 여러가지 "해결경로"를 생성하고, 이를 탐색 및 평가하여 최적의 답을 찾는 기법
	2. 하나의 선형적인 흐름이 아닌 나무 구조처럼 여러 가능성을 열어 두고 'Self-Evaluation(자기평가)'을 통해 유망한 경로를 선택하거나 막다른 길에서 되돌아 옴(BackTracking)
	3. (장점) 창의적인 문제 해결이나 매우 복잡한 전략적 판단이 필요한 작업에 유리
	4. (단점) 다른 기법에 비해 프롬프트 구조가 복잡하고 연산 비용이 매우 높음


| 구분     | CoT          | ReAct                 | ToT                 |
| ------ | ------------ | --------------------- | ------------------- |
| 주요 특징  | 논리적 단계 나열    | 추론과 외부 도구 활용 결합       | 다중 경로 탐색 및 평가       |
| 작동 방식  | 선형적(A->B->C) | 순환적(Reason->Act->Obs) | 계층적(Tree structure) |
| 적합 분야  | 수학 문제, 상식 추론 | 정보 검색, 실시간 데이터 활용     | 전략 수립, 창의적 문제 해결    |
| 핵심 키워드 | "단계별로 생각하기"  | "검색하고 실행하기"           | "모든 가능성 검토하기"       |

# 이 세가지 기법은 상호 베타적이 아닌 필요에 따라 혼합하여 사용되기도 함
예) ReAct 구조 내에서 각 추론 단계를 CoT 방식으로 서술
