#Overfitting  #ML #L1 #L2

머신러닝에서 정규화(Reqularization)는 모델이 훈련 데이터에 너무 과하게 적합되어 실제 데이터에서 성능이 떨어지는 과적합(Overfitting) 문제를 해결하기 위한 기법

학습 시 손실 함수(Loss Function)에 모델의 가중치(Weight) 크기에 따른 '패널티'를 추가하여 모델이 너무 복잡해지지 않도록 가중치를 조절하는 원리

# 1. L1  정규화(Lasso Regularization)
- 가중치의 절대값의 합을 패널티로 부여
- 수식적 특징 : 손실 함수 뒤에 𝝀𝚺 | 𝘸 | 이 붙음 (𝝀 람다는 패널티의 강도를 조절하는 [^1]하이퍼파라미터)
- 어떤 가중치 (𝘸) 값들을 정확히 0으로 만듬
- (장점) 영량력이 적은 특정(Feature)들을 제거하는 효과가 있어 특성 선택(Feature Selection) 기능이 포함된 셈, 모델이 간결해져 해석력이 좋아짐

[^1]: 머신러닝 모델을 학습시킬때 사용자가 직접 설정해 주는 값

# 2. L2 정규화(Ridge Regularization)
- 가중치의 제곱의 합을 패널티로 부여
- 수식적 특징: 손실 함수 뒤에 𝝀𝚺𝘸² 이 붙음
- 가중치 (𝘸) 값을 작게(0에 가깝게) 만듬, 하지만 L1처럼 0으로 만들지는 않음
- (장점) 특정 가중치가 너무 커지는 것을 방지하여 모델의 변동성을 줄이고 안정성을 높임, 일반적으로 성능면에서 L1보다 더 널리 사용딤


# 요약
-  데이터에 불필요한 변수가 많고 핵심적인 변수만 골라내고 싶을때 : L1(Lasso)가 유리
* 데이터의 모든 변수가 중요하거나, 전반적인 에측 성능의 안정성을 높이고 싶을때 : L2(Ridge)가 유리
* 두 가지를 섞어 쓰는 것을 **Elastic Net**이라고 함
