#ChainOfThought #프롬프트엔지니어링 

기존의 일반적은 프롬프트 방식(Few-shot)은 질문과 정답의 쌍만 보여주지만
CoT는 "질문"-"추론과정"-"정답"의 구조를 보여줌

# 주요유형
* Few-Shot CoT
	* 사용자가 프롬프트에 추론 과정이 포함된 예시를 몇개 직접 작성해서 보여주는 방법
* Zero-Shot CoT*
	* 예스를 들 필요도 없이 "단계별로 차근차근 생각해봐(Let's think step by step)"이라는 문구만 추가하는 방식

# 한계점
* 추론 비용 증가 : 중간 과정을 생성하는 만큼 출력 토큰 수가 늘어나 비용과 시간이 더 소요
* 논리 오류의 전파 : 중간 추론 단계에서 계산 실수나 논리 비약이 발생하면, 최종 정답도 틀리게 됨
	* 이를 보완하기 위해 여러 개의 추론 경로를 만들고 투표하는 Self-Consistency 기법 등이 함께 쓰임
