#API_서빙 #API_Serving

### 1) 개념(Concept)

**API 서빙(API Serving)**은 개발된 AI 모델을 실제 서비스 환경에 배포하여, 외부 프로그램(앱, 웹 등)이 네트워크를 통해 모델에 데이터를 보내고 예측 결과(추론)를 받아갈 수 있게 만드는 과정입니다. 모델이라는 '엔진'을 자동차에 실어 실제로 도로 위를 달릴 수 있게 하는 '출시' 단계와 같습니다.

---

### 2) 왜 필요한가(Background & Problem Solving)

- **독립된 환경:** AI 모델은 특수한 라이브러리(PyTorch, TensorFlow)와 고성능 하드웨어(GPU)가 필요합니다. 이를 일반 웹 서버와 분리하여 독립적으로 관리하기 위해 필요합니다.
    
- **실시간 응답:** 사용자가 입력한 데이터를 즉시 처리하여 결과를 돌려주어야 합니다.
    
- **확장성(Scalability):** 사용자가 1명일 때와 100만 명일 때, 서버가 자동으로 늘어나거나 줄어들며 대응해야 합니다.
    

---

### 3) 어떻게 동작하는가(Mechanism)

일반적으로 **REST API** 방식을 사용하며, 데이터 전달 형식으로는 **JSON**을 주로 사용합니다.

1. **요청(Request):** 클라이언트가 텍스트나 이미지 데이터를 API 서버로 보냅니다.
    
2. **전처리(Preprocessing):** 서버 내에서 데이터를 모델이 이해할 수 있는 형태(토큰화, 정규화 등)로 변환합니다.
    
3. **추론(Inference):** 로드된 AI 모델에 데이터를 넣어 결과값을 계산합니다.
    
4. **후처리(Postprocessing):** 계산된 숫자 데이터를 인간이 읽을 수 있는 텍스트나 카테고리로 변환합니다.
    
5. **응답(Response):** 최종 결과를 클라이언트에게 다시 전송합니다.
    

---

### 4) 수학적 관점: 지연 시간(Latency)

API 서빙의 성능을 평가할 때 가장 중요한 지표는 **응답 속도(Latency)**입니다.

$$L_{total} = T_{network} + T_{preprocess} + T_{inference} + T_{postprocess}$$

- **① 수식 전체 의미:** 사용자가 요청을 보낸 후 결과를 받을 때까지 걸리는 전체 시간은 각 단계별 소요 시간의 합입니다.
    
- **② 각 기호의 의미:**
    
    - $L_{total}$: 전체 지연 시간(Total Latency)
        
    - $T_{network}$: 데이터가 인터넷을 통해 오가는 시간
        
    - $T_{preprocess}$: 토큰화, 정규화 등 데이터 준비 시간
        
    - $T_{inference}$: 모델이 실제 계산을 수행하는 시간
        
    - $T_{postprocess}$: 결과값을 정리하는 시간
        
- **③ 실제 계산 방식:** 네트워크 20ms, 전처리 10ms, 추론 100ms, 후처리 5ms라면 전체 응답 속도는 135ms가 됩니다.
    
- **④ 왜 필요한가:** 실무에서는 사용자 경험을 위해 $L_{total}$을 최소화해야 하며, 어느 구간에서 병목(Bottleneck)이 발생하는지 파악하여 최적화하기 위해 이 수식이 필요합니다.
    

---

### 5) 실무 및 인프라 연결

- **Docker & Kubernetes:** AI 모델과 실행 환경을 **컨테이너(Docker)**로 패키징하고, 이를 **쿠버네티스(Kubernetes)**를 통해 관리하며 트래픽에 따라 서버 대수를 조절합니다.
    
- **MLOps:** 모델이 서빙되는 동안 예측이 정확한지 감시(Monitoring)하고, 성능이 떨어지면 다시 학습시키는 전체 주기를 관리합니다.
    
- **GPU 서빙:** **LLM**처럼 거대한 모델은 매우 빠른 계산이 필요하므로, **NVIDIA Triton Inference Server**와 같은 전문 엔진을 사용하여 GPU 자원을 최적화합니다.
    

---

### 6) 어디에 쓰이는가(Applications)

- **LLM 서비스:** ChatGPT처럼 질문을 던지면 답변을 주는 API가 대표적입니다.
    
- **RAG 시스템:** 사용자의 질문을 받아 **벡터DB**에서 정보를 검색하고 결과를 조합하여 응답할 때 API 서빙 구조가 활용됩니다.
    

---

### 7) 이해를 돕는 비유

- **초등학생 수준:** 식당의 '주문 창구'와 같습니다. 손님이 메뉴(데이터)를 말하면 주방(모델)에서 음식을 만들어 창구를 통해 내어주는 과정 전체를 말합니다.
    
- **실무자 수준:** 학습된 가중치(Weights) 파일과 전처리 로직을 HTTP 프로토콜로 노출하여, 이기종 시스템 간 상호운용성(Interoperability)을 확보하는 **모델 배포(Deployment)** 단계입니다.
