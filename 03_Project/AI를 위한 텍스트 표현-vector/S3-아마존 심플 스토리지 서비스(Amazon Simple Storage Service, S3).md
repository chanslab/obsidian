#S3 

### 1) 개념(Concept)

**S3(Simple Storage Service)**는 아마존 웹 서비스(AWS)에서 제공하는 **객체 스토리지(Object Storage)** 서비스입니다. 인터넷 어디서나 원하는 양의 데이터를 저장하고 검색할 수 있도록 구축된 '온라인 데이터 저장소'입니다. 컴퓨터의 하드디스크나 USB 메모리와 비슷하지만, 물리적 장치가 아닌 클라우드 환경에 존재하며 거의 무한한 확장성을 가집니다.

---

### 2) 왜 필요한가(Background & Problem Solving)

AI 프로젝트, 특히 **말뭉치(Corpus)**를 다룰 때는 다음과 같은 문제에 직면합니다.

- **용량 문제:** LLM 학습을 위한 데이터는 수 테라바이트(TB)에서 페타바이트(PB)에 달하여 일반 서버의 하드디스크로는 감당하기 어렵습니다.
    
- **내구성 문제:** 하드디스크는 고장 날 위험이 있습니다. 데이터가 유실되면 막대한 학습 비용이 낭비됩니다.
    
- **접근성 문제:** 여러 대의 학습 서버(GPU 노드)가 동시에 같은 데이터에 접근해야 합니다.
    

**S3(Simple Storage Service)**는 데이터를 여러 물리적 위치에 복제하여 저장함으로써 **99.999999999%(11개의 9)의 내구성(Durability)**을 보장하며, 수만 대의 서버가 동시에 데이터를 읽을 수 있는 환경을 제공하여 이 문제들을 해결합니다.

---

### 3) 어떻게 동작하는가(Mechanism)

S3는 파일 시스템 구조가 아닌 **객체(Object)** 단위로 데이터를 관리합니다.

1. **버킷(Bucket):** 데이터를 담는 최상위 컨테이너(폴더와 유사한 개념)입니다. 이름은 전 세계에서 유일해야 합니다.
    
2. **객체(Object):** 실제 데이터 파일입니다. 여기에는 데이터 본체와 이를 설명하는 **메타데이터(Metadata)**가 포함됩니다.
    
3. **키(Key):** 객체를 찾기 위한 고유한 이름(경로)입니다.
    

---

### 4) 어디에 쓰이는가(Applications)

- **데이터 레이크(Data Lake):** 정제되지 않은 방대한 **말뭉치(Corpus)**를 쌓아두는 중앙 저장소로 사용됩니다.
    
- **모델 체크포인트(Model Checkpoint) 저장:** LLM 학습 중 몇 일마다 생성되는 모델의 중간 상태(가중치 데이터)를 안전하게 보관합니다.
    
- **정적 웹 사이트 호스팅:** 전처리가 완료된 데이터 결과물을 API 형태로 제공하기 전 단계의 저장소로 쓰입니다.
    

---

### 5) 실무 및 인프라 관점 (MLOps & RAG)

실제 시스템 아키텍처에서 S3는 다음과 같이 배치됩니다.

- **MLOps 파이프라인:** **쿠버네티스(Kubernetes)** 환경에서 동작하는 학습 컨테이너들이 S3를 데이터 소스로 바라봅니다. 데이터가 업데이트되면 이를 감지하여 자동으로 재학습을 시작하는 트리거 역할을 수행하기도 합니다.
    
- **RAG 구조:** 원본 문서(PDF, 텍스트 등)를 S3에 저장해두고, **임베딩(Embedding)** 과정을 거쳐 **벡터DB(Vector Database)**로 보낼 때의 원천 데이터 저장소로 활용됩니다.
    

---

### 6) 이해를 돕는 비유

- **초등학생 수준:** 전 세계 누구든 파일을 넣을 수 있고, 아무리 많이 넣어도 꽉 차지 않는 '마법의 거대 가방'입니다. 가방에 이름표(Key)를 붙여두면 언제든 다시 꺼낼 수 있습니다.
    
- **실무자 수준:** HTTP 프로토콜 기반의 REST API를 통해 접근 가능한 고가용성 분산 객체 저장소입니다. 데이터 수집(Ingestion) 단계에서 **데이터 레이크(Data Lake)**의 기반 인프라로 작동합니다.

[다음 단계 학습]

- **데이터 레이크(Data Lake):** S3와 같은 저장소에 모인 방대한 데이터를 어떻게 체계적으로 관리하고 분석할 것인지에 대한 전략입니다.
    
- **IAM(Identity and Access Management):** S3에 저장된 민감한 AI 학습 데이터에 누가 접근할 수 있는지 권한을 제어하는 보안 체계입니다.
    
- **데이터 전처리 파이프라인(ETL):** S3에 저장된 가공되지 않은 텍스트를 AI 모델이 학습할 수 있는 형태로 변환하는 흐름입니다.
