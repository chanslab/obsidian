#토큰화 #Tokenization

### 1) 개념(Concept)

**토큰화(Tokenization)**는 방대한 **말뭉치(Corpus)**를 분석하기 위해, 텍스트를 **토큰(Token)**이라 불리는 더 작은 단위로 잘게 나누는 과정입니다. <mark style="background: #FFB8EBA6;">여기서 토큰은 문장, 단어, 형태소, 또는 문자(Character) 단위가 될 수 있으며, </mark>AI가 언어를 이해하는 가장 기초적인 데이터 단위가 됩니다.

---

### 2) 왜 필요한가(Background & Problem Solving)

- **컴퓨터의 한계:** 컴퓨터는 "사과를 먹었다"라는 문장 전체를 하나의 개념으로 직접 이해할 수 없습니다. 데이터를 처리하기 위해서는 이를 숫자로 바꾸어야 하는데, 그전에 처리 가능한 최소 단위로 쪼개는 작업이 선행되어야 합니다.
    
- **어휘 사전(Vocabulary) 관리:** 세상의 모든 문장을 다 저장할 수는 없지만, 자주 쓰이는 단어들을 모은 **어휘 사전(Vocabulary)**을 만들면 효율적으로 언어를 관리할 수 있습니다.
    
- <mark style="background: #BBFABBA6;">**신조어 및 복합어 처리:**</mark> "열공(열심히 공부)"과 같은 단어를 "열"과 "공"으로 쪼개어 학습하면, 모델이 처음 보는 단어라도 그 구성 요소를 통해 의미를 유추할 수 있습니다.
    

---

### 3) 어떻게 동작하는가(Mechanism)

토큰화는 크게 세 가지 방식으로 구분됩니다.

| **방식**                      | **설명**                  | **장점**                          | **단점**                            |
| --------------------------- | ----------------------- | ------------------------------- | --------------------------------- |
| **단어 단위(Word-based)**       | 공백을 기준으로 단어를 분리         | 직관적이고 구현이 쉬움                    | 어휘 사전이 너무 커짐 (사과, 사과를, 사과는 각각 저장) |
| **문자 단위(Character-based)**  | 'ㄱ', 'ㄴ', 'ㅏ' 등 글자별로 분리 | 모든 단어를 표현 가능(OOV 문제 해결)         | 토큰 배열이 길어져 계산량이 급증함               |
| **부분 단어 단위(Subword-based)** | 통계적으로 자주 등장하는 글자 조합 추출  | **LLM**에서 주로 사용(BPE, WordPiece) | 적절한 토큰 결합 알고리즘이 필요함               |
 * OOV : Out of Vocaburaly
---

### 4) 수학적 관점: 토큰 매핑(Token Mapping)

토큰화 과정은 텍스트 집합($\mathcal{T}$)을 고유한 정수 인덱스 집합($\mathcal{I}$)으로 변환하는 함수 $f$로 표현할 수 있습니다.

$$f: \mathcal{T} \rightarrow \{i_1, i_2, \dots, i_n\}, \quad \text{where } i \in \{1, 2, \dots, |V|\}$$

- **① 수식 전체 의미:** 텍스트 문장을 입력받아, **어휘 사전(Vocabulary)** 내의 인덱스 번호들의 나열(Sequence)로 변환한다는 의미입니다.
    
- **② 각 기호의 의미:**
    
    - $f$: 토큰화 함수(Tokenizer)
        
    - $\mathcal{T}$: 원본 텍스트(Text)
        
    - $i_n$: 각 토큰에 부여된 고유 번호(Index)
        
    - $|V|$: 어휘 사전의 전체 크기(Vocabulary Size)
        
- **③ 실제 계산 방식:** 사전 $V$가 `{ "나는": 1, "학교에": 2, "간다": 3 }`일 때, "나는 학교에 간다"라는 문장은 $[1, 2, 3]$이라는 정수 리스트로 변환됩니다.
    
- **④ 왜 필요한가:** 딥러닝 모델의 연산은 행렬 계산(Matrix Multiplication)이므로, 텍스트를 행렬 연산이 가능한 '숫자 인덱스'로 바꾸어야만 모델 입력이 가능합니다.
    

---

### 5) 어디에 쓰이는가(Applications)

- **거대언어모델(LLM):** GPT-4와 같은 모델은 **부분 단어 토큰화(Subword Tokenization)**를 사용하여 수만 개의 토큰으로 수조 개의 문장을 처리합니다.
    
- **검색 엔진:** 사용자의 검색어를 토큰으로 나누어 관련 문서를 찾습니다.
    

---

### 6) 실무 및 인프라 관점

- **API 서빙(Serving):** **LLM** 서비스를 배포할 때, 클라이언트가 보낸 원문(Raw Text)을 서버가 받자마자 가장 먼저 수행하는 작업이 토큰화입니다. <mark style="background: #ABF7F7A6;">이 과정에서 발생하는 속도가 전체 응답 속도(Latency)에 영향을 줍니다.</mark>
    
- **MLOps:** 학습 시 사용한 **토큰화 알고리즘(Tokenizer)** 파일은 모델 가중치 파일과 함께 **Docker** 이미지나 **S3**에 반드시 같이 보관되어야 합니다. 모델이 학습할 때 썼던 사전과 추론할 때 쓰는 사전이 다르면 엉뚱한 결과가 나오기 때문입니다.
    

---

### 7) 이해를 돕는 비유

- **초등학생 수준:** 레고 성(문장)을 부수어서 각각의 블록(토큰)으로 나누는 것과 같습니다. 블록마다 번호를 붙여두면, 나중에 번호만 보고도 성을 다시 조립하거나 분석할 수 있습니다.
    
- **실무자 수준:** 연속적인 텍스트 데이터를 불연속적인 이산적 벡터 공간(Discrete Vector Space)의 인덱스로 변환하는 **전처리(Preprocessing)** 단계입니다.

- **텍스트 정규화(Text Normalization):** "먹었다", "먹다"를 하나의 표준 형태로 바꾸어 학습 효율을 높이는 과정입니다.
    
- **불용어 제거(Stopword Removal):** "은, 는, 이, 가"처럼 의미가 적은 단어를 제거하여 연산 비용을 줄이는 기술입니다.
    
- **패딩(Padding):** 서로 다른 길이의 토큰 문장들을 일정한 크기의 행렬로 만들기 위해 빈 공간을 0으로 채우는 기법입니다.
